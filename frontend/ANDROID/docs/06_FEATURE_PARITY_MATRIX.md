- **Employee auth + reset**: feature/employee-auth and feature/password-reset already call /api/v1/portal/login and /api/v1/portal/reset, store tokens via DataStore, and display Compose forms with validation.
- **Employee attendance**: feature/employee-attendance fetches /api/v1/attendance per month, shows month navigation, keeps an offline queue in core/data/AttendanceQueueRepository, and implements the TEĎ punch button/configurable stats consistent with the web utilities.
- **Time input + calculations**: core/domain/TimeInputNormalizer and AttendanceCalculator port the web logic (HHMM normalizer, weekday/holiday detection, employee template behavior, break windows, afternoon/weekend tallies) and are covered by unit tests.
- **Admin shell + features**: Framework modules/network endpoints exist (core/network/AdminApi), but Compose screens/VMs for users, attendance, shift plan, export, prints, and settings are still TODO; the navigation graph currently exposes only employee routes.
- **Deep links**: /app, /reset?token=..., /admin/login, /admin/users, /admin/dochazka, /admin/plan-sluzeb, /admin/export, /admin/tisky, and /admin/settings are documented; only the first two are wired today.
- **Update distribution**: Play flavor pulls from Play Core, direct flavor expects /android-update.json (see core/update and ANDROID/docs/08_UPDATE_DISTRIBUTION_SPEC.md), but backend metadata and APK hosting need coordination.
